# Express 数据库集成

Web 应用的核心价值在于数据的持久化与查询。Express 本身不提供数据库功能，但可以与主流数据库无缝集成。本章将系统学习 MongoDB 和 MySQL 在 Express 中的使用方法。

## MongoDB 与 Mongoose

MongoDB 是文档型 NoSQL 数据库，以灵活的 JSON-like 文档存储数据，非常适合快速迭代和 schema 多变的场景。

### 安装依赖

```bash
npm install mongoose
```

### 连接数据库

```js
// config/database.js
const mongoose = require('mongoose');

const connectDB = async () => {
  try {
    const conn = await mongoose.connect(process.env.MONGO_URI, {
      // Mongoose 6+ 默认启用这些选项，显式写出更清晰
      // useNewUrlParser: true,
      // useUnifiedTopology: true,
    });
    console.log(`MongoDB 已连接: ${conn.connection.host}`);
  } catch (err) {
    console.error('数据库连接失败:', err.message);
    process.exit(1);
  }
};

module.exports = connectDB;
```

```js
// app.js
const connectDB = require('./config/database');

connectDB();
app.listen(3000, () => console.log('服务启动'));
```

### 定义 Schema 与 Model

```js
// models/User.js
const mongoose = require('mongoose');

const userSchema = new mongoose.Schema({
  name: {
    type: String,
    required: [true, '姓名不能为空'],
    trim: true,
    maxlength: [50, '姓名不能超过 50 个字符']
  },
  email: {
    type: String,
    required: true,
    unique: true,
    lowercase: true
  },
  role: {
    type: String,
    enum: ['user', 'admin'],
    default: 'user'
  },
  createdAt: {
    type: Date,
    default: Date.now
  }
});

// 虚拟字段（不存入数据库）
userSchema.virtual('domain').get(function () {
  return this.email.split('@')[1];
});

// 实例方法
userSchema.methods.greet = function () {
  return `你好，${this.name}`;
};

// 静态方法
userSchema.statics.findByEmail = function (email) {
  return this.findOne({ email });
};

module.exports = mongoose.model('User', userSchema);
```

### CRUD 操作

```js
// controllers/userController.js
const User = require('../models/User');

// 创建
exports.createUser = async (req, res) => {
  const user = await User.create(req.body);
  res.status(201).json(user);
};

// 查询列表
exports.getUsers = async (req, res) => {
  const users = await User.find().select('-__v');
  res.json(users);
};

// 查询单个
exports.getUser = async (req, res) => {
  const user = await User.findById(req.params.id);
  if (!user) {
    return res.status(404).json({ error: '用户不存在' });
  }
  res.json(user);
};

// 更新
exports.updateUser = async (req, res) => {
  const user = await User.findByIdAndUpdate(
    req.params.id,
    req.body,
    { new: true, runValidators: true }
  );
  res.json(user);
};

// 删除
exports.deleteUser = async (req, res) => {
  await User.findByIdAndDelete(req.params.id);
  res.status(204).send();
};
```


## MySQL 与 mysql2

MySQL 是经典的关系型数据库，适合数据结构化程度高、事务要求严格的场景。

### 安装依赖

```bash
npm install mysql2
```

### 连接与连接池

```js
// config/database.js
const mysql = require('mysql2/promise');

const pool = mysql.createPool({
  host: process.env.DB_HOST || 'localhost',
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
});

module.exports = pool;
```

> 使用**连接池**（connection pool）比每次创建新连接更高效，连接池会自动管理连接的复用和释放。

### 执行查询

```js
// controllers/userController.js
const pool = require('../config/database');

exports.getUsers = async (req, res) => {
  const [rows] = await pool.execute('SELECT id, name, email FROM users');
  res.json(rows);
};

exports.getUser = async (req, res) => {
  const [rows] = await pool.execute(
    'SELECT id, name, email FROM users WHERE id = ?',
    [req.params.id]
  );
  if (rows.length === 0) {
    return res.status(404).json({ error: '用户不存在' });
  }
  res.json(rows[0]);
};

exports.createUser = async (req, res) => {
  const { name, email } = req.body;
  const [result] = await pool.execute(
    'INSERT INTO users (name, email) VALUES (?, ?)',
    [name, email]
  );
  res.status(201).json({ id: result.insertId, name, email });
};
```

### 预处理语句防注入

mysql2 的 `execute()` 方法使用预处理语句，自动转义参数，从根本上防止 SQL 注入：

```js
// ✅ 安全：参数被正确转义
const [rows] = await pool.execute(
  'SELECT * FROM users WHERE email = ? AND status = ?',
  [email, status]
);

// ❌ 危险：直接拼接字符串
const [rows] = await pool.execute(
  `SELECT * FROM users WHERE email = '${email}'`
);
```

## ORM 选型：Prisma

对于大型项目，直接使用原始 SQL 或 Mongoose 的链式 API 可能不够灵活。**Prisma** 是一个现代化的 ORM，同时支持 PostgreSQL、MySQL、SQLite、SQL Server 和 MongoDB。

### Prisma 快速上手

```bash
npm install prisma @prisma/client
npx prisma init
```

```prisma
// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "mysql"
  url      = env("DATABASE_URL")
}

model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String
  createdAt DateTime @default(now())
  posts     Post[]
}

model Post {
  id       Int    @id @default(autoincrement())
  title    String
  content  String
  authorId Int
  author   User   @relation(fields: [authorId], references: [id])
}
```

```js
// 使用 Prisma Client
const { PrismaClient } = require('@prisma/client');
const prisma = new PrismaClient();

const users = await prisma.user.findMany({
  include: { posts: true },
  where: { email: { contains: '@example.com' } }
});
```

## 数据库选型对比

| 维度 | MongoDB | MySQL |
|------|---------|-------|
| 数据模型 | 文档型（灵活 Schema） | 关系型（严格 Schema） |
| 查询语言 | JSON-like 查询 | SQL |
| 事务支持 | 4.0+ 支持多文档事务 | 原生强事务支持 |
| 扩展性 | 水平扩展友好 | 垂直扩展为主 |
| 适用场景 | 快速原型、内容管理、日志 | 金融、电商、ERP |

## 连接池管理最佳实践

```js
// 应用关闭时优雅释放连接
process.on('SIGINT', async () => {
  await pool.end();
  console.log('数据库连接已关闭');
  process.exit(0);
});
```

## 小结

本章覆盖了 Express 应用中数据库集成的核心技术：

- **Mongoose**：Schema 定义、验证、虚拟字段、方法扩展、关联查询
- **mysql2**：Promise 接口、预处理语句、连接池管理
- **Prisma ORM**：跨数据库的现代化 ORM 方案
- **安全实践**：始终使用参数化查询防止 SQL 注入
- **选型原则**：根据数据结构化程度和事务需求选择合适的数据库

为 API 接入数据库后，应用才真正具备了业务价值。下一章将聚焦于认证与安全，学习如何保护数据不被未授权访问。
